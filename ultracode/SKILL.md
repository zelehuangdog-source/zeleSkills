# Ultracode 多智能体编排器

当用户需要通过多 agent 并行协作来完成复杂任务时，使用本 skill 进行结构化编排。本 skill 封装了从需求澄清到方案确认到执行的完整流程。

> **触发词**：ultracode、多智能体、workflow、并行 agent、编排

---

## 流程（三步走，严格按顺序执行，禁止跳步）

### Phase 1：需求澄清

调用 `/clarify` skill，传入用户的 $ARGUMENTS，让 clarify 自行完成需求澄清。澄清完毕后会输出需求摘要。

---

### Phase 2：方案设计 & 用户确认

基于澄清后的需求，设计 Workflow 脚本。**写完脚本后，不要立即执行**，而是向用户详细讲解：

#### 必须讲清楚的内容：

1. **整体架构**
   - 共几个 phase？每个 phase 做什么？
   - 并行几个 agent？各自的角色和职责？

2. **每个 agent 的 prompt 要点**
   - 不需要贴完整 prompt，但要说清楚每个 agent 的输入是什么、期望输出是什么
   - 如果 agent 之间有依赖关系（前一阶段输出作为后一阶段输入），要画清楚数据流

3. **汇总/评估策略**
   - 最终结果怎么汇总？
   - 如果有评估 agent，评估的维度和标准是什么？

4. **预期产出**
   - 最终会输出什么给用户？
   - 格式是什么样的？

#### 讲解格式（参考）：

```markdown
## Workflow 方案

### 整体设计
- Phase 1: {描述}（{N} 个 agent 并行）
- Phase 2: {描述}（{M} 个 agent）

### Agent 角色

| # | 角色 | 输入 | 输出 | 备注 |
|---|---|---|---|---|
| 1~N | {角色名} | {输入描述} | {输出描述} | 并行/串行 |
| N+1 | {评估/汇总} | 前 N 个 agent 的输出 | {最终产出} | |

### 数据流
{用文字或简图说明 agent 之间的数据流转}

### 预期产出
{最终输出的形式和内容}
```

讲解完后，使用 AskUserQuestion 问用户：

- 方案是否符合预期？
- 需要调整什么？（agent 数量、prompt 内容、评估维度等）

**用户明确确认后，才进入 Phase 3。**

---

### Phase 3：执行

**🔥 禁止自己手动执行任务逻辑（Bash 查询、文件操作等）。Phase 3 的唯一动作是调用 Workflow 工具。**

即使用户在 Phase 2 确认时补充了新信息（如"要真实执行查询"），也只能将这些信息融入 Workflow 脚本的 agent prompt 中，由 subagent 执行，绝不能自己动手跑。探查路径、验证环境等前置操作应在 Phase 2 完成。

调用 Workflow 工具执行脚本。执行期间：

- 告知用户已启动，可用 `/workflows` 查看进度
- Workflow 完成后，将最终结果（汇总报告/评估结果/改进建议等）完整呈现给用户
- 如果 workflow 失败或部分 agent 失败，如实告知并分析原因

---

## 铁律

1. **Phase 1 → 2 → 3 严格顺序**，不可跳步，不可合并
2. **Phase 2 必须等用户确认**才能执行，禁止自动开跑
3. **Workflow 脚本不要写文件**，直接在 Workflow 工具的 script 参数中传入
4. **每个 agent 的 prompt 必须自包含**，agent 之间不共享上下文
5. **不要吞错误**：agent 返回 null 时要在最终报告中说明
6. **数据量大时注意 substring 截断**：传给评估 agent 的内容如果太长，要合理截断并说明
7. **Phase 3 禁止手动执行**：用户确认方案后，唯一允许的工具调用是 `Workflow`。禁止在 Phase 3 用 Bash/Read/Write 等工具自行执行任务逻辑（探查 skill 路径、验证环境等前置操作应在 Phase 2 完成）

---

## 输入

$ARGUMENTS
