# sync-frontend

后端设计完 API 后，一键生成结构化前端提示词，消除跨 session 信息传递的损耗。

## 解决的痛点

当你用两个 Claude Code session 分别负责后端和前端时，后端设计完 API 后需要把方案传递给前端：

### 1. 占用后端 agent 工作时间

后端 session 的核心职责是设计 API、写业务逻辑。但每次设计完，你都要让它停下来"帮我总结一下给前端"——它开始回顾对话、组织格式、输出文档。这个过程动辄几十秒到一分钟，期间你只能干等着。更糟的是，如果你对总结不满意，还要来回修改。

后端 agent 本该专注于后端的事，"整理文档给前端"不是它的职责，却反复占用它的工作时间。

### 2. 污染后端 agent 上下文

让后端 agent 做总结意味着它的上下文窗口里多了一大段"我来帮你整理前端需要的信息……"的输出。这些内容对后端后续的工作毫无价值，却实实在在地占据了宝贵的 token 空间。

上下文越长，模型的注意力越分散。你在后端 session 里做完同步后继续改代码，会发现它的表现微妙地变差了——因为它的"工作记忆"被无关内容稀释了。

### 3. 需要手动复制、组装提示词给前端

即使后端总结好了，你还要自己完成最后一公里：复制内容、切换到前端窗口、粘贴、可能还要加一句"请按这个接口实现"。如果总结里没有明确告诉前端"你该做什么"（先澄清？先设计 UI？直接写代码？），前端 agent 收到后还会反问你，又一轮沟通。

整个流程：后端总结 → 你复制 → 你粘贴 → 你补充指令 → 前端开始干活。中间全是人力胶水。

## 用户故事

> 我在后端 session 里花了 20 分钟设计耦合检测的 3 个 API。设计完了，输入 `/fork /sync-frontend`，然后直接让后端 agent 开始写代码——**后端 agent 没有被打断，上下文里也没有多出任何"帮你整理前端文档"的垃圾内容**。
>
> 1 分钟后浏览器弹出预览页面，我扫了一眼：3 个接口的 Request/Response 结构、枚举值、边界说明全部提取到位。更关键的是，**它不只是一份接口文档，而是一段组装好的完整提示词**——包含需求背景、API 契约、以及明确的任务指令（先 /clarify 澄清 → 再出 HTML 原型 → 最后实现代码），前端 agent 收到后知道该做什么、按什么顺序做。
>
> 切到前端 session，**Cmd+V 回车**——剪贴板里已经自动复制好了一句启动指令，不需要我手动从文档里摘内容、不需要额外写一句"请按这个接口实现"。前端 agent 直接读取文件，按步骤开始工作。
>
> 以前：让后端总结（等 30 秒）→ 手动复制 → 粘贴到前端 → 补一句指令 → 前端反问"枚举值有哪些"。现在：一个命令，**提示词自动组装，剪贴板自动就绪**，粘贴即用。

## 正确用法

在后端 session 中，API 设计讨论完毕后，输入：

```
/fork /sync-frontend
```

然后切到前端 session，Cmd+V 粘贴即可。

## 它做了什么

`/fork` 派生一个继承完整对话上下文的后台子 agent，子 agent 执行 `/sync-frontend` 指令，整个过程不占用后端主 session 的时间和上下文。具体步骤：

1. **提取 API 契约**：从当前对话中自动提取所有接口的完整定义——URL、Method、Request/Response 结构、枚举值、错误码、边界说明
2. **询问前端行为模式**：让你选择前端收到后应该怎么做（支持多选，按顺序串联执行）：

| 模式 | 说明 |
|------|------|
| 先澄清 | 前端先用 /clarify 确认理解无误再动手 |
| 设计示例 HTML UI 界面 | 前端先出一个纯 HTML 页面骨架，你确认后再写正式代码 |
| 直接实现 | 跳过澄清和设计，直接写 TypeScript 类型 + API hook + 页面组件 |

3. **组装结构化提示词**：根据你的选择，生成一段完整的、可直接驱动前端 session 工作的 prompt（包含需求背景 + API 契约 + 明确的任务指令）
4. **保存到 Obsidian**：落盘到 `~/Documents/obsidian_repo/工程迭代/{工程名}/{需求名}/sync-frontend-{日期}.md`，留档可追溯
5. **生成预览 HTML 并打开**：浏览器弹出预览页面，你可以当场 review 完整内容
6. **复制到剪贴板**：一句 `读取 {文件路径} 并按其中的任务要求执行`，粘贴给前端即可驱动工作

## 依赖

- Claude Code v2.1.32+（需要 `/fork` 命令支持）
- macOS（使用 `pbcopy` 和 `open`）
- [clarify](../clarify/) skill（前端"先澄清"模式依赖）

## 安装

将 `sync-frontend/` 目录复制到 `~/.claude/skills/` 下：

```bash
cp -r sync-frontend ~/.claude/skills/
```
